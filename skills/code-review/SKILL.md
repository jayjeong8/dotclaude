# 코드 리뷰

## 실행 절차

1. `git diff --staged`로 스테이징된 변경사항 확인 (없으면 `git diff`로 전체 변경사항 확인)
2. 수정된 파일에 집중하여 아래 체크리스트 기반으로 리뷰
3. 문제 발견 시 구체적인 수정 예시를 포함하여 피드백

## 체크리스트

### 코드 품질
- 코드가 간결하고 읽기 쉬운가
- 함수와 변수 이름이 적절한가
- 중복 코드가 없는가
- 적절한 에러 처리가 되어 있는가

### 보안
- 노출된 비밀키나 API 키가 없는가
- 입력값 검증이 구현되어 있는가

### 테스트 및 성능
- 테스트 커버리지가 충분한가
- 성능 고려사항이 반영되어 있는가

## 피드백 우선순위
- 심각 (반드시 수정)
- 경고 (수정 권장)
- 제안 (개선 고려)

# 단계별 리팩토링 실행

제공된 계획을 한 단계씩 실행합니다. 절대 여러 단계를 한 번에 처리하지 마세요.

## 실행 규칙

각 단계마다 다음 순서를 반드시 따르세요:

1. **구현**: 해당 단계의 코드 변경만 수행
2. **린트**: 수정한 파일에 대해 린트 실행, 에러 수정
3. **커밋**: `/commit` 스킬로 해당 단계만의 원자적 커밋 생성
4. **다음 단계**: 이전 단계가 커밋된 후에만 다음 단계 진행

## 금지사항

- 여러 단계를 하나의 커밋으로 묶기
- 모든 코드 변경을 먼저 하고 나중에 커밋하기
- 린트 실패를 무시하고 커밋하기

## 에러 처리

- 린트/타입체크 실패 시: 해당 단계 내에서 수정 후 커밋
- 2회 이상 수정 시도 실패 시: 사용자에게 보고하고 중단

# 도메인 지식 조회

## 질문 유형별 검색 우선순위

| 질문 유형 | 1순위 | 2순위 | 3순위 |
|-----------|-------|-------|-------|
| 일반 도메인 질문 | Slack | Linear | Confluence |
| 이슈/버그 관련 질문 | Linear | Slack | Confluence |
| 공식 스펙/문서 질문 | Confluence | Linear | Slack |

## 실행 절차

### 1단계: 질문 분석
- 사용자 질문에서 핵심 주제와 의도 파악
- 위 표에서 해당하는 질문 유형 결정
- 검색에 사용할 키워드 2~5개 추출 (한국어/영어 모두)

### 2단계: 우선순위별 검색 실행
- 결정된 우선순위에 따라 순서대로 검색
- 각 소스별 검색은 같은 폴더의 가이드 참조:
  - Slack 검색: `search-slack.md`
  - Linear 검색: `search-linear.md`
  - Confluence 검색: `search-confluence.md`
- 1순위에서 충분한 정보를 얻으면 2, 3순위는 보충 확인만 수행
- 1순위에서 부족하면 2, 3순위를 적극 활용

### 3단계: 정보 종합 및 답변
- 수집된 정보를 시간순 또는 중요도순으로 정리
- 상충하는 정보가 있으면 최신 정보 우선 + 차이점 명시

## 출력 형식

```
## 답변
[질문에 대한 종합 답변]

## 출처
- [소스 유형] 출처 설명 (날짜)
  - 예: [Slack] #channel-name - @작성자의 메시지 (2025-01-15)
  - 예: [Linear] TEAM-123 이슈 제목 (2025-02-01)
  - 예: [Confluence] 문서 제목 (URL)

## 신뢰도
[높음/중간/낮음] - 판단 근거
```

## 신뢰도 판단 기준
- **높음**: 공식 문서 또는 최근 3개월 내 복수 소스에서 일관된 정보
- **중간**: 단일 소스이거나, 3개월 이상 경과한 정보
- **낮음**: 간접적 정보만 있거나, 소스 간 상충하는 내용 존재

## 주의사항
- 찾을 수 없는 정보는 "확인되지 않음"으로 명시 (추측 금지)
- 검색 결과가 없으면 어떤 소스를 검색했는지 + 대안 제시
- 민감 정보(토큰, 비밀번호 등)가 검색 결과에 포함되면 마스킹 처리
- 사용자에게 추가 컨텍스트(채널명, 이슈 번호 등)를 요청하면 검색 정확도 향상

# 디버깅 프로세스

## 단계
1. 에러 메시지와 스택 트레이스 수집
2. 재현 단계 파악
3. 실패 지점 격리
4. 최소한의 수정 구현
5. 해결책 검증

## 분석 방법
- 에러 메시지와 로그 분석
- 최근 코드 변경사항 확인
- 가설 수립 및 테스트
- 전략적 디버그 로깅 추가
- 변수 상태 검사

## 결과 제공 항목
- 근본 원인 설명
- 진단을 뒷받침하는 증거
- 구체적인 코드 수정
- 테스트 방법
- 재발 방지 권장사항

## 참고
- TypeScript 타입 에러인 경우 `/fix-types` 스킬도 활용

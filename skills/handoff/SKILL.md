# Handoff 문서 생성

현재 작업 상태를 분석하여 다음 세션에서 이어서 작업할 수 있는 구조화된 핸드오프 문서를 생성합니다.

## 실행 절차

### 1단계: 현재 상태 수집

병렬로 다음 정보를 수집:

```bash
# 현재 브랜치
git branch --show-current

# 커밋 히스토리 (base 브랜치로부터)
git log --oneline origin/main..HEAD 2>/dev/null || git log --oneline -20

# 미커밋 변경사항
git status --short
git diff --stat
```

### 2단계: 관련 문서 확인

- `.claude/specs/` 디렉토리에서 현재 브랜치/이슈와 관련된 문서 검색
- 브랜치명에 이슈 ID가 있으면 해당 spec 파일 확인

### 3단계: 핸드오프 문서 생성

다음 구조로 작성:

```markdown
# Handoff: {브랜치명}

**날짜**: YYYY-MM-DD
**이슈**: [이슈 ID] (있는 경우)

## 컨텍스트
- **목표**: [이 브랜치에서 달성하려는 것]
- **관련 스펙**: [spec 파일 경로] (있는 경우)

## 완료된 작업
커밋 목록 (최신순):
1. `abc1234` feat: 기능 A 구현
2. `def5678` refactor: 기존 코드 정리

## 진행 중인 작업
미커밋 변경사항:
- `src/file.ts` - [변경 내용 요약]
- `src/other.ts` - [변경 내용 요약]

## 남은 작업
- [ ] 작업 1
- [ ] 작업 2
- [ ] 테스트 작성

## 핵심 결정사항
- [결정 1]: [이유]
- [결정 2]: [이유]

## 다음 세션 시작 프롬프트
```
[브랜치명]에서 작업을 이어서 해줘.
.claude/handoffs/[파일명].md를 읽고 컨텍스트를 파악한 뒤 남은 작업을 진행해.
```
```

### 4단계: 저장

1. `.claude/handoffs/` 디렉토리 생성 (없으면)
2. `{branch-name}.md` 파일명으로 저장
3. 같은 브랜치의 기존 핸드오프가 있으면 덮어쓰기

### 5단계: 결과 보고

```
✅ 핸드오프 문서가 생성되었습니다.
📄 .claude/handoffs/{branch-name}.md

다음 세션에서 위 파일을 참고하여 작업을 이어갈 수 있습니다.
```

## 주의사항

- 커밋 메시지와 diff에서 작업 내용을 자동 추론
- "남은 작업"은 spec 파일이나 이슈에서 인수 기준을 참고하여 작성
- 핵심 결정사항은 현재 대화에서 중요한 기술적 결정을 요약
- 민감 정보(토큰, 비밀번호 등)가 포함되지 않도록 주의

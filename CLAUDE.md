## 워크플로우 규칙

### 원자적 커밋 (최우선 규칙)
멀티 스텝 계획을 실행할 때, 반드시 각 단계마다 별도로 커밋하세요. 여러 변경을 하나의 커밋으로 묶지 마세요. 계획에 번호가 매겨진 단계가 있으면, 각 단계 = 하나의 원자적 커밋입니다.

### 린트 → 커밋
코드 수정 후 커밋 전에 항상 수정된 파일에 대해 린트를 실행하세요. 린트 에러가 있으면 커밋 전에 수정하세요.

## Git & 커밋
- 커밋 메시지는 별도 지시가 없으면 한국어로 작성
- 커밋 시 `/commit` 스킬 사용
